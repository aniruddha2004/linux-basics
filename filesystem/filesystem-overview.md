# Linux Filesystem Hierarchy

> Overview of the standard Linux directory structure and what each directory is meant for.

---

## The Root Directory `/`

All directories in Linux branch from a single root. There are no drive letters like in Windows. Everything — files, devices, processes, network sockets — lives somewhere under `/`.

---

## Core Directories

### `/bin` — Essential User Binaries
Programs needed to boot the system and for basic recovery. Available to all users.

```bash
ls /bin    # cat, cp, ls, mv, rm, mkdir, bash, ps, kill
```

> These must work even when `/usr` is not mounted.

---

### `/sbin` — System Binaries
Administrative commands for system maintenance. Typically requires root.

```bash
ls /sbin   # fdisk, mkfs, reboot, shutdown, iptables, ifconfig
```

---

### `/boot` — Boot Loader Files
Files needed to start Linux: kernel, initramfs, and bootloader configuration.

```bash
ls /boot   # vmlinuz-6.x, initrd.img, grub/
```

> Keep this on a simple filesystem. Often mounted separately.

---

### `/dev` — Device Files
Special files representing hardware and virtual devices. Accessing these files talks to the kernel drivers.

```bash
ls /dev    # sda (disk), tty (terminal), null, zero, random, stdin, stdout
```

> Not real files on disk — generated by the kernel.

---

### `/etc` — Configuration Files
System-wide configuration. Text files that control how programs and services behave.

```bash
ls /etc    # passwd, shadow, fstab, hosts, apt/sources.list, ssh/sshd_config
```

> Editable by root. Programs read these at startup.

---

### `/home` — User Home Directories
Personal space for each user. Contains documents, downloads, settings, and user-specific data.

```bash
ls /home   # aniruddha_ghosh, test-user
```

> Users have full control inside their own home. Other users cannot access without permission.

---

### `/lib` and `/lib64` — Essential Shared Libraries
Dynamic libraries required by programs in `/bin` and `/sbin`. The 64-bit variant is `/lib64`.

```bash
ls /lib    # libc.so, libm.so, ld-linux.so
```

> Like `.dll` files in Windows. Programs link to these at runtime.

---

### `/media` — Removable Media Mount Points
Where USB drives, CDs, and external disks are automatically mounted by the desktop environment.

```bash
ls /media  # USB_DRIVE, CDROM
```

---

### `/mnt` — Temporary Mount Points
Where administrators manually mount filesystems temporarily. Empty by default.

```bash
sudo mount /dev/sdb1 /mnt/backup
```

> Use `/media` for automatic mounts, `/mnt` for manual ones.

---

### `/opt` — Optional Software
Third-party or standalone applications installed outside the package manager.

```bash
ls /opt    # google/chrome, microsoft/teams, some-vendor-app
```

> Self-contained software bundles go here.

---

### `/proc` — Process Information
Virtual filesystem exposing kernel data as files. Covered in depth in `../process-management/proc-filesystem.md`.

```bash
ls /proc   # 1, 72131, 65513, cpuinfo, meminfo, uptime
```

> Nothing here is stored on disk. Generated live by the kernel.

---

### `/root` — Root User's Home
The home directory for the root (superuser) account. Not located under `/home` for security.

```bash
ls /root   # (only accessible as root)
```

> Kept separate so root can log in and fix things even if `/home` is corrupted or unmounted.

---

### `/tmp` — Temporary Files
Space for programs to create short-lived temporary files. Cleared on reboot (usually).

```bash
ls /tmp    # session files, downloads-in-progress, lock files
```

> Anyone can write here. Files older than a few days are typically auto-deleted.

---

### `/usr` — User Programs and Data
The largest directory. Contains most user-accessible programs, libraries, and documentation.

```bash
ls /usr
├── bin     # Most user commands (gcc, python, vim, htop)
├── sbin    # Non-essential system binaries
├── lib     # Libraries for /usr/bin and /usr/sbin
├── local   # Locally compiled/installed software
├── share   # Architecture-independent data (docs, icons, fonts)
└── include # Header files for compiling C programs
```

> Originally "user" binaries, now it's the main software repository. Can be shared across machines via network.

---

### `/var` — Variable Data
Files that change constantly during normal operation: logs, caches, spools, databases.

```bash
ls /var
├── log     # System and application logs (syslog, auth.log, nginx/access.log)
├── cache   # Cached application data
├── spool   # Queued tasks (mail, print jobs, cron)
├── tmp     # Persistent temp files (survives reboot unlike /tmp)
└── lib     # Variable state for applications (databases, package lists)
```

> Logs grow here. Monitoring disk usage in `/var/log` is important.

## Summary Table

| Directory | Purpose |
|-----------|---------|
| `/bin` | Essential user commands |
| `/sbin` | System administration commands |
| `/boot` | Boot loader and kernel files |

| `/etc` | System configuration files |
| `/home` | User home directories |
| `/lib` / `/lib64` | Essential shared libraries |

| `/usr` | Majority of user programs and data |

## Dotfiles in Home Directory

When a new user is created, their home directory is populated with **hidden configuration files** (dotfiles) from `/etc/skel/`.

### What Gets Copied

```bash
ls -a /home/aniruddha_ghosh
.bash_logout  .bashrc  .profile  .config/  .local/
```

| File | Source | Purpose |
|------|--------|---------|
| `.bashrc` | `/etc/skel/.bashrc` | Shell configuration loaded for **interactive** sessions. Aliases, prompts, functions |
| `.bash_logout` | `/etc/skel/.bash_logout` | Commands run when logging out of a bash shell |
| `.profile` | `/etc/skel/.profile` | Shell configuration loaded at **login**. Sets PATH, runs startup programs |

### Where They Come From

```bash
cat /etc/skel/.bashrc
```

The `/etc/skel/` directory contains template files. When `useradd -m` creates a home directory, it copies everything from `/etc/skel/` into the new home.

> Administrators can customize `/etc/skel/` to give new users company-standard aliases, PATH settings, or welcome messages.

### What Each File Does

**`.bashrc`**
- Runs every time you open a new terminal window
- Defines aliases (`ll='ls -la'`), custom prompts, shell functions
- Loads completion scripts
- Safe to modify freely

**`.profile`**
- Runs once when you log in (SSH, GUI login, console)
- Sets environment variables like `PATH`, `EDITOR`, `LANG`
- Often sources `.bashrc` to ensure terminal settings are loaded
- Runs before `.bashrc` in login shells

**`.bash_logout`**
- Runs when you exit a login shell
- Can clean up temp files or log activity
- Less commonly customized

## Key Distinctions

| Question | Answer |
|----------|--------|
| `/bin` vs `/usr/bin` | `/bin` is for boot-critical commands. `/usr/bin` is everything else |
| `/lib` vs `/usr/lib` | Same relationship — `/lib` for boot, `/usr/lib` for the rest |
| `/tmp` vs `/var/tmp` | `/tmp` cleared on reboot. `/var/tmp` persists |
| `/media` vs `/mnt` | `/media` for automatic mounts. `/mnt` for manual admin mounts |
| `/etc` vs `/usr/local/etc` | `/etc` for system packages. `/usr/local/etc` for manually installed software |
